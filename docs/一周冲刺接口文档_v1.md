## 2. 全局约定

## 2.1 路径前缀

- 统一前缀：`/api/v1`
- 说明：与项目开发流程最小清单保持一致，便于后续版本扩展。

## 2.2 统一响应格式

```json
{
  "code": 1,
  "msg": "success",
  "data": {}
}
```

- `code=1`：成功
- `code=0`：失败
- 当前项目建议统一返回 HTTP 200，业务结果通过 `code/msg` 区分

## 2.3 鉴权规则

- 公开接口：高校、校区、POI 查询接口
- 需要登录（Bearer Token）：`/users/me`、`POST /wishes`、`DELETE /wishes/{wishId}`
- 请求头：

```http
Authorization: Bearer <token>
```

## 2.4 时间与命名格式

- 时间：`yyyy-MM-dd HH:mm:ss` 或 ISO 字符串（二选一，项目内保持统一）
- 路径参数命名：`universityId`、`campusId`、`poiId`、`wishId`
- JSON 字段命名：统一驼峰，如 `officialWebsite`、`suggestedDuration`

## 4. 本周新增模块 1：JWT 认证闭环

## 4.1 用户注册

- 方法：`POST`
- 路径：`/api/v1/auth/register`
- 鉴权：否
- 请求体（JSON）：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `username` | `String` | 是 | 用户名，建议 4-20 位 |
| `password` | `String` | 是 | 明文入参，入库前必须 BCrypt |
| `nickname` | `String` | 否 | 昵称 |
| `identityType` | `Integer` | 否 | 1高中生/2大学生 |
| `highSchool` | `String` | 否 | 毕业高中 |
| `targetUniId` | `Long` | 否 | 目标高校 |
| `currentUniId` | `Long` | 否 | 当前就读高校 |

请求示例：

```json
{
  "username": "xiaochen",
  "password": "12345678",
  "nickname": "小陈",
  "identityType": 2,
  "highSchool": "北京市第四中学",
  "targetUniId": 1,
  "currentUniId": null
}
```

成功响应示例：

```json
{
  "code": 1,
  "msg": "success",
  "data": null
}
```

失败场景建议：

- 用户名已存在 -> `code=0, msg=用户名已存在`
- 参数不合法 -> `code=0, msg=参数错误`

## 4.2 用户登录

- 方法：`POST`
- 路径：`/api/v1/auth/login`
- 鉴权：否
- 请求体：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `username` | `String` | 是 | 用户名 |
| `password` | `String` | 是 | 密码 |

请求示例：

```json
{
  "username": "xiaochen",
  "password": "12345678"
}
```

成功响应 `data` 示例：

```json
{
  "token": "eyJhbGciOiJIUzI1NiJ9.xxx.yyy",
  "tokenType": "Bearer",
  "expiresIn": 7200
}
```

失败场景建议：

- 用户不存在/密码错误 -> `code=0, msg=用户名或密码错误`
- 用户被禁用 -> `code=0, msg=账号已禁用`

## 4.3 获取当前用户

- 方法：`GET`
- 路径：`/api/v1/users/me`
- 鉴权：是

成功响应 `data` 示例：

```json
{
  "id": 1,
  "username": "xiaochen",
  "nickname": "小陈",
  "role": 0,
  "identityType": 2,
  "highSchool": "北京市第四中学",
  "targetUniId": 1,
  "currentUniId": null
}
```

失败场景建议：

- Token 缺失/非法/过期 -> `code=0, msg=未登录或登录已过期`

## 5. 本周新增模块 2：愿望墙 CRUD

## 5.1 建议表结构（最小）

表名：`ugc_wish`

```sql
CREATE TABLE IF NOT EXISTS ugc_wish (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  uni_id BIGINT NOT NULL,
  content VARCHAR(100) NOT NULL,
  color VARCHAR(10) DEFAULT '#FFD966',
  create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  is_deleted TINYINT NOT NULL DEFAULT 0
);
```

## 5.2 发布愿望

- 方法：`POST`
- 路径：`/api/v1/wishes`
- 鉴权：是
- 请求体：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `universityId` | `Long` | 是 | 目标高校ID |
| `content` | `String` | 是 | 愿望内容，建议 1-100 字 |
| `color` | `String` | 否 | HEX 颜色，如 `#FFD966` |

请求示例：

```json
{
  "universityId": 1,
  "content": "明年一定上岸北邮！",
  "color": "#FFD966"
}
```

成功响应示例：

```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "wishId": 1001
  }
}
```

## 5.3 按高校分页查询愿望

- 方法：`GET`
- 路径：`/api/v1/universities/{universityId}/wishes`
- 鉴权：否
- 路径参数：
  - `universityId`：`Long`，必填
- 查询参数：
  - `page`：`Integer`，默认 `1`
  - `size`：`Integer`，默认 `10`

成功响应 `data` 示例：

```json
{
  "total": 23,
  "page": 1,
  "size": 10,
  "list": [
    {
      "id": 1001,
      "userId": 1,
      "universityId": 1,
      "content": "明年一定上岸北邮！",
      "color": "#FFD966",
      "createTime": "2026-04-16 21:30:00"
    }
  ]
}
```

## 5.4 删除愿望（本人逻辑删除）

- 方法：`DELETE`
- 路径：`/api/v1/wishes/{wishId}`
- 鉴权：是
- 路径参数：
  - `wishId`：`Long`，必填
- 删除规则：
  - 只允许本人删除
  - 执行逻辑删除（`is_deleted = 1`）

成功响应示例：

```json
{
  "code": 1,
  "msg": "success",
  "data": null
}
```

失败场景建议：

- 愿望不存在 -> `code=0, msg=愿望不存在`
- 非本人删除 -> `code=0, msg=无权限删除该愿望`

## 6. 本周新增模块 3：POI 查询增强（实现约束）

## 6.1 分类映射

- `1`：文化
- `2`：学术
- `3`：生活
- `4`：风景

## 6.2 SQL 行为约束

- 必须默认过滤：`is_deleted = 0`
- 当传 `category` 时追加条件：`AND category = #{category}`
- 排序规则根据 `sort` 控制，默认 `hot_score DESC`

## 6.3 返回类型约束

- 返回必须是数组：`PoiVO[]` / `List<PoiVO>`
- 不允许单对象返回（避免多行查询异常）
